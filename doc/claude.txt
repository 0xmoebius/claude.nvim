==============================================================================
1. INTRODUCTION                                          *claude-introduction*

claude.nvim is an alternative frontend for the Claude Code CLI. You get a
cross-project session picker, per-session vim tabs, a real buffer for the
prompt, live streaming of assistant responses into a markdown buffer,
gutter-bar role markers, and a status bar showing the current context
percentage, git branch, and optional subscription usage.

The plugin shells out to `claude -p --output-format stream-json` under the
hood; it reads and writes the same ~/.claude/projects/ JSONL store as the
official CLI, so sessions are shared across both.


==============================================================================
2. REQUIREMENTS                                          *claude-requirements*

- Neovim ≥ 0.10
- `claude` CLI on $PATH
- Python 3 (only for the permission-forwarding hook; stdlib only)
- Optional: telescope.nvim, oil.nvim or neo-tree.nvim


==============================================================================
3. QUICK START                                            *claude-quick-start*

Install the plugin (see README for plugin-manager snippets), then:
>
    require("claude").setup({})
<
Symlink the launcher onto PATH: >
    ln -s /path/to/claude.nvim/bin/cc ~/.local/bin/cc
<
Run `cc` in any shell. You'll get a picker across every Claude Code
session on your machine. Pick one, nvim opens in that session's cwd with
the transcript and prompt wired up.

4. COMMANDS                                                  *claude-commands*

                                                                      *:Claude*
:Claude             Open the cross-project session picker. Alias for
                    |claude.launch|.

                                                                  *:ClaudePick*
:ClaudePick         Same picker; when you pick a session already open in a
                    tab, jumps to it instead of duplicating.

                                                                   *:ClaudeNew*
:ClaudeNew          Start a new session in the current cwd. Opens in a new
                    tab.

                                                                  *:ClaudeSend*
:ClaudeSend         Send the prompt buffer contents. Also bound to <CR> in
                    normal mode inside the prompt buffer.

                                                             *:ClaudeInterrupt*
:ClaudeInterrupt    SIGINT the current turn's subprocess.

                                                             *:ClaudeYankBlock*
:ClaudeYankBlock    Yank the fenced code block under cursor (without the
                    backticks) to the `+` register.

                                                              *:ClaudeYankLast*
:ClaudeYankLast     Yank the most recent assistant reply to `+`.

                                                                  *:ClaudeQuit*
:ClaudeQuit         Close all Claude tabs, wipe buffers, kill in-flight
                    jobs, and quit nvim if no other tabs remain.


==============================================================================
5. KEYMAPS                                                    *claude-keymaps*

All keymaps are buffer-local to Claude panes and configurable via
|claude-config|. Defaults:
>
    <CR>          prompt → send (normal mode)
    <C-c>         prompt → interrupt turn
    <leader>ca    focus prompt (insert mode)
    <leader>ct    focus transcript
    i/a/o/I/A/O   transcript → focus prompt (read-only redirect)
    ]m / [m       transcript → next / prev message marker
    <leader>cs    picker (new tab)
    <leader>cn    new session in cwd
    <leader>cc    close this Claude tab
    <leader>cq    quit: close all + exit nvim
    <leader>cy    yank last assistant reply
    gy            transcript → yank fenced code block
    gt / gT       switch between tabs (vim native)
<


==============================================================================
6. CONFIGURATION                                               *claude-config*

Set any option by passing it to |claude.setup|. The defaults are listed in
the README; a minimal override example: >

    require("claude").setup({
      permission_mode = "bypassPermissions",
      subscription_usage = true,
      keymaps = {
        send = "<S-CR>",    -- use Shift-Enter instead of plain Enter
        pick = false,       -- disable the picker keymap
      },
      layout = {
        files_width = 0.2,
        prompt_height = 0.25,
      },
    })
<

7. HIGHLIGHTS                                              *claude-highlights*

claude.nvim defines the following highlight groups (all created with
`default = true`, so your colorscheme wins):
>
    ClaudeUserSign       → DiagnosticOk     (your message bar)
    ClaudeAssistantSign  → Function         (claude's reply bar)
    ClaudeToolSign       → DiagnosticWarn   (tool calls/results)
    ClaudeErrorSign      → DiagnosticError  (errors)
    ClaudeTab            → TabLine          (tabline, inactive tab)
    ClaudeTabSel         → TabLineSel       (tabline, active tab)
<
Override directly: >
    vim.api.nvim_set_hl(0, "ClaudeAssistantSign", { fg = "#cba6f7" })
<


==============================================================================
8. HOW IT WORKS                                             *claude-internals*

Per turn, the plugin spawns `claude -p` with stream-json output and pipes
your prompt to stdin. Stdout is parsed line-by-line; events become calls
into the renderer via `vim.schedule`. Session identity is the JSONL file's
UUID; continuing a session is just `--resume <uuid>`.

State lives per-tab in |state.tabs|, keyed by tab handle. Each record holds
the session id, cwd, buffer handles, in-flight job, last observed usage,
history ring, and tab-scoped always-allow list for permissions.

9. SUBSCRIPTION USAGE                              *claude-subscription-usage*

`subscription_usage = true` (default) shows the 5-hour quota usage in the
winbar. This uses the UNDOCUMENTED endpoint:
>
    POST https://api.anthropic.com/api/oauth/usage
<
with your Claude Code OAuth token read from the macOS keychain (or
`~/.claude/.credentials.json`). Results are cached at
`~/.cache/claude.nvim/usage.json` for 180s and rate-limited to one request
per 30s, with 429 backoff honored. Set to false if:

- You don't want an undocumented endpoint touched.
- You'd prefer the plugin not read your OAuth token at runtime (it flows
  only into the curl call to api.anthropic.com).

Use `:ClaudeUsageDebug` to inspect the fetch state.

10. INTERACTIVE PERMISSIONS                                *claude-permissions*

Set `ask_permissions = true` to route tool permissions into nvim:
>
    ask_permissions = true,
    permission_tools = { "Bash", "Write", "Edit" },
    permission_always_allow = { "Read", "Grep", "Glob" },
<
On any listed tool, Claude pauses until you pick Yes / No / Always via
`vim.fn.confirm()`. "Always" is scoped to the current tab.

Mechanics: the plugin passes a PreToolUse hook in a one-off `--settings`
JSON blob to `claude -p`. The hook invokes `bin/claude-nvim-auth` (Python),
which calls back into nvim via `--remote-expr` on |v:servername|. The
return value travels back to Claude as the hook's permissionDecision.

The same --settings blob ALWAYS includes an AskUserQuestion matcher
(independent of `ask_permissions`); see |claude-questions|.

11. INTERACTIVE QUESTIONS (AskUserQuestion)              *claude-questions*

When a skill calls the `AskUserQuestion` tool (e.g. `/fma`, failure-mode
analysis), the plugin opens a floating picker instead of letting the CLI
short-circuit with an error.

Keybindings:
    j, k / Up, Down     navigate options
    <Space>             toggle current option (multi-select only)
    <CR>                confirm
    <Esc>, q, <C-c>     cancel (reports "user dismissed" to Claude)

The float shows the question at the top, options in the middle, and a
preview pane with the focused option's full description. No search bar.

No configuration required. The hook is wired unconditionally.

Mechanics: same PreToolUse hook script as permissions, dispatching on
`tool_name == "AskUserQuestion"` to `require("claude.questions").start`.
The picker can't run inside a synchronous `--remote-expr` (the TUI
freezes), so the hook uses a tempfile handoff: nvim schedules the
picker and writes the answer to disk; the hook polls until it appears
and forwards the text as the tool_result via deny+reason.

12. TROUBLESHOOTING                                   *claude-troubleshooting*

`:echo v:servername`
                    Should be non-empty. The plugin calls |serverstart()|
                    on first launch; if your init explicitly prevents
                    sockets, permissions forwarding can't work.

`:messages`
                    All errors from async handlers land here.

`:ClaudeInterrupt`
                    Cancel a stuck turn. The job is SIGINTed; state is
                    cleared.

Picker shows no sessions
                    Check `ls ~/.claude/projects/*/*.jsonl`. If the
                    directory is empty, run `claude` at least once to
                    create a session.

5h shows "—"
                    Either `subscription_usage = false`, or the OAuth
                    fetch failed (no token, 429, or network error). Check
                    `~/.cache/claude.nvim/usage.json`.
